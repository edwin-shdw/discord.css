Embeds have a colored border, an author field, thumbnail, title, description, embedded images, text fields and a footer.
Discord.css uses [grid](https://developer.mozilla.org/en-US/docs/Web/CSS/grid) under the hood to arrange the components.


<CodePreview bordered={true}>
  <EmbedExample />
</CodePreview>

```html
<div class="dc-embed" style="--dc-embed-color: #a00; line-height: normal;">
    <div class="dc-embed-author">
        <img src="/img/dcs.webp" alt="Author image">
        <span>discord.css</span>
    </div>
    <div class="dc-embed-thumbnail">
        <img src="/img/dcs.webp" alt="Thumbnail">
    </div>
    <div class="dc-embed-title">Embeds</div>
    <div class="dc-embed-description">
        If you create a bot you will probably using embeds, so this class is ideal if you want to build
        previews for the dashboard of your bot!
    </div>
    <div class="dc-embed-fields">
        <div class="dc-embed-field-inline">
            <div class="dc-embed-field-name">Field name</div>
            <div class="dc-embed-field-value">Some field value</div>
        </div>
        <div class="dc-embed-field-inline">
            <div class="dc-embed-field-name">Inline field name</div>
            <div class="dc-embed-field-value">Some other field value</div>
        </div>
        <div class="dc-embed-field">
            <div class="dc-embed-field-name">Field name</div>
            <div class="dc-embed-field-value">No longer inline field</div>
        </div>
    </div>
    <div class="dc-embed-image"><img src="/img/shadow-logo.png" alt="Shadow"></div>
    <div class="dc-embed-footer">
        <img src="/img/dcs.webp" alt="Author Profile Image">
        <span>
            discord.css docs
            <span class="dc-embed-footer-seperator">•</span>
            30.10.2023 22:13
        </span>
    </div>
</div>
```

## Color

You can customize the border color by setting a color on the `--dc-embed-color` variable or the `border-color` property.
It's up to you which one you use both will result in the same embed.

<CodePreview bordered={true}>
  <Embed color="#f0f">
    <Embed.Title>Setting colors</Embed.Title>
  </Embed>
</CodePreview>

```html
<div class="dc-embed" style="--dc-embed-color: #f0f;">
    <div class="dc-embed-title">Setting colors</div>
</div>
```

```html
<div class="dc-embed" style="border-color: #f0f;">
    <div class="dc-embed-title">Setting colors</div>
</div>
```


## Author

Author fields contain an author image and name.

<CodePreview bordered={true}>
  <Embed>
    <Embed.Author>
      <img src="/img/dcs.webp" />
      <span>Some name</span>
    </Embed.Author>
  </Embed>
</CodePreview>

```html
<div class="dc-embed">
    <div class="dc-embed-author">
        <img src="/img/dcs.webp">
        <span>Some name</span>
    </div>
</div>
```

Author fields can also contain links inside them.

<CodePreview bordered={true}>
  <Embed>
    <Embed.Author>
      <img src="/img/dcs.webp" />
      <a href="#author">Some name</a>
    </Embed.Author>
  </Embed>
</CodePreview>

```html
<div class="dc-embed">
    <div class="dc-embed-author">
        <img src="/img/dcs.webp">
        <a href="#author">Some name</a>
    </div>
</div>
```

```html
    <div class="dc-embed-thumbnail">
        <img src="/img/dcs.webp">

```

:::warning
Discord.css uses the `:has` selector for embeds. Firefox does [not support](https://caniuse.com/css-has) this selector
yet so it will appear slightly different on Firefox browsers.
:::

## Fields

Fields contain a pair of field name and field value. Fields can be displayed either as a block or inline which will
result in a row of fields.

<CodePreview bordered={true}>
  <Embed>
    <Embed.Fields>
      <Embed.Field>
        <Embed.Field.Name>Field 1</Embed.Field.Name>
        <Embed.Field.Value>Will be displayed as a block</Embed.Field.Value>
      </Embed.Field>
      <Embed.Field inline={true}>
        <Embed.Field.Name>Field 2</Embed.Field.Name>
        <Embed.Field.Value>Will be displayed inline next to field 3</Embed.Field.Value>
      </Embed.Field>
      <Embed.Field inline={true}>
        <Embed.Field.Name>Field 3</Embed.Field.Name>
        <Embed.Field.Value>Will be displayed inline next to field 2</Embed.Field.Value>
      </Embed.Field>
    </Embed.Fields>
  </Embed>
</CodePreview>

```html
<div class="dc-embed">
    <div class="dc-embed-fields">
        <div class="dc-embed-field">
            <div class="dc-embed-field-name">Field 1</div>
            <div class="dc-embed-field-value">Will be displayed as a block</div>
        </div>
        <div class="dc-embed-field-inline">
            <div class="dc-embed-field-name">Field 2</div>
            <div class="dc-embed-field-value">Will be displayed inline next to field 3</div>
        </div>
        <div class="dc-embed-field-inline">
            <div class="dc-embed-field-name">Field 3</div>
            <div class="dc-embed-field-value">Will be displayed inline next to field 2</div>
        </div>
    </div>
</div>
```

:::note
Discord will display a maximum of three fields per row.
:::

## Footer

The footer consists of an image, some text and a timestamp. The timestamp is seperated by a `•` and should be wrapped by
a `<span>` with the `dc-embed-footer-seperator` class.

<CodePreview bordered={true}>
  <Embed>
    <Embed.Footer>
      <img src="/img/dcs.webp" />
      <span>DCS Docs<span className="dc-embed-footer-seperator">•</span>30.10.2023 22:13</span>
    </Embed.Footer>
  </Embed>
</CodePreview>

```html
<div class="dc-embed">
    <div class="dc-embed-footer">
        <img src="/img/dcs.webp">
        <span>
            DCS Docs
            <span class="dc-embed-footer-seperator">•</span>
            30.10.2023 22:13
        </span>
    </div>
</div>
```


## Notes and limits

If you want your embeds as realistic as possible you should follow the notes and Discords API limitations.

### Notes
- To display fields side-by-side, you need at least two consecutive fields set to `inline`
- The timestamp should be adjusted to the timezone depending on the user's device
- Mentions of any kind in embeds will only render correctly within embed descriptions and field values
- Embeds allow masked links (e.g. `[GitHub](https://github.com/)`), but only in description and field values

### Limits

- Embed titles are limited to 256 characters
- Embed descriptions are limited to 4096 characters
- There can be up to 25 fields
- A field's name is limited to 256 characters and its value to 1024 characters
- The footer text is limited to 2048 characters
- The author name is limited to 256 characters
- The sum of all characters from all embed structures in a message must not exceed 6000 characters
